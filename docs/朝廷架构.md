## 📋 任务自动记录机制

### 工具位置
`/root/clawd/silijian/task_recorder.py`

### 分派流程
```
1. 皇上旨意 → 司礼监接收
2. 司礼监调用 task_recorder.create_task() 创建政务系统记录
3. 司礼监调用 sessions_spawn 分派给六部
4. 司礼监调用 task_tracker.track_task() 开始追踪
5. 六部执行完成 → completion event 触发
6. task_tracker.complete_task() 自动更新状态
7. 司礼监回奏皇上
```

### 使用示例
```python
from task_recorder import create_task, update_task_status
from task_tracker import track_task

# 创建任务
task = create_task(
    title="修复后端 API 500 错误",
    description="检查后端服务状态、日志和数据库连接",
    agent_id="gongbu",
    priority="high"
)

# 分派并追踪
result = sessions_spawn(agentId="gongbu", task="...")
track_task(result["childSessionKey"], task["task_id"])

# 完成后自动更新（completion event 触发）
```

## 📊 政务系统 API

### 任务管理
- `POST /api/tasks/` - 创建任务
- `GET /api/tasks/` - 获取任务列表
- `PUT /api/tasks/{id}` - 更新任务
- `GET /api/tasks/{id}/flows` - 获取流转历史

### 流转记录
- `POST /api/tasks/{id}/flows` - 记录流转
- `GET /api/flows/task/{id}` - 获取流转时间线

### 大臣配置
- `GET /api/ministers/` - 获取大臣列表
- `GET /api/ministers/openclaw/config` - 从 openclaw.json 读取
- `POST /api/ministers/openclaw/reload` - 重新加载配置
- `PUT /api/ministers/openclaw/sync` - 同步修改到 openclaw.json
